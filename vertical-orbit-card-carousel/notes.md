# Vertical Orbit Card Carousel

## Primary Use Case

A premium stacked-card carousel where multiple visual cards rotate in a smooth vertical circular motion around an invisible elliptical axis.

Designed to create a cinematic editorial showcase where one card becomes visually dominant while others support depth, anticipation, and motion hierarchy.

This animation is built for premium hero sections and high-trust visual introductions.

---

## Best Used For

- premium SaaS landing pages
- creator intro sections
- product showcase reveals
- portfolio hero sections
- editorial design showcases
- authority-building homepage intros
- premium service presentation sections
- cinematic visual storytelling blocks

Especially strong when presenting multiple high-value items in a single premium motion system.

---

## Visual Behavior

Cards move in a continuous vertical orbital loop.

The system simulates an invisible rotating wheel:

- front cards appear larger
- rear cards appear smaller
- front cards remain sharper
- rear cards become slightly blurred
- opacity increases toward the front
- z-depth changes reinforce dimensionality
- subtle rotation offsets prevent flatness

The motion feels expensive because movement is slow, restrained, and intentional.

The carousel should never feel like a loud slider.

It should feel like a controlled product presentation.

---

## Motion Sequence

1. Initial stacked card layout appears centered

2. Cards begin moving along a vertical elliptical path

3. Front-most card becomes visually dominant through:
   - increased scale
   - reduced blur
   - stronger shadow presence
   - improved opacity
   - stronger z-index

4. Rear cards move behind the visual hierarchy

5. Continuous looping maintains premium ambient motion

6. Subtle floating motion on the entire container prevents mechanical stiffness

The goal is cinematic continuity rather than obvious carousel behavior.

## Avoid Using When

Avoid using when:

- the section is heavily text-dependent
- information density is extremely high
- the user must compare detailed metrics quickly
- the layout needs fast practical utility over premium presentation
- visual hierarchy should remain fully static

This is a premium presentation animation, not a dashboard utility component.
